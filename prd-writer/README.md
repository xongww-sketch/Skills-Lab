# 📋 PRD Writer — 产品需求文档生成器

一套面向 AI Agent 的 PRD 写作 Skill，从需求描述/聊天记录自动生成**清晰、可执行、研发友好**的产品需求文档。

## 🎯 能做什么

| 能力 | 说明 | 典型场景 |
|---|---|---|
| 需求整理 | 从聊天记录/口头描述提取核心要素 | "把这段对话整理成PRD" |
| 文档撰写 | 按标准结构输出PRD，多表多图少废话 | "给研发写个方案文档" |
| 流程图生成 | 自动生成彩色流程图/架构图/泳道图 | 插入飞书文档，不用ASCII |
| 飞书输出 | 直接创建飞书文档，图片插入正确位置 | 一键交付，不用手动排版 |

## 🚀 快速开始

把 `prd-writer/` 目录放到你的 OpenClaw workspace 的 `skills/` 下即可自动识别。

然后对 AI 说：
- "帮我写个PRD"
- "把这个需求整理成产品文档给研发"
- "写一份技术方案文档"

## 📁 目录结构

```
prd-writer/
├── SKILL.md                          # Skill 定义（AI 读这个）
├── README.md                         # 人类读这个
└── references/
    ├── diagram-style.md              # 流程图/架构图 HTML 样式规范（色板+CSS）
    └── example-structure.md          # PRD 章节结构模板（基于实战案例）
```

## 📐 核心原则

1. **表格 > 文字** — 能用表格说清的绝不写段落
2. **图片 > 代码块** — 流程图必须生成图片，不用 ASCII
3. **研发视角** — 写具体实现，不写商业价值
4. **异常必覆盖** — 每个流程都要有失败处理方案
5. **数据示例** — 用真实格式举例，不写抽象描述

## 🎨 流程图风格

自动生成的流程图采用彩色卡片式设计：

| 颜色 | 用途 |
|---|---|
| 🔵 蓝色 | 数据源/取数 |
| 🟢 绿色 | 匹配/处理 |
| 🟠 橙色 | 分批/拆分/通知 |
| 🟣 紫色 | 生成/输出 |
| 🔴 红色 | 外部系统/RPA |
| ⚫ 灰色 | 日志/记录 |

## 📋 适用场景

- 新功能/新模块的产品需求文档
- 系统对接方案（API/RPA/第三方）
- 数据同步/迁移方案
- 任何需要给研发团队看的技术文档

## ❌ 不适用

- 事故复盘报告（用专门的复盘模板）
- 会议纪要整理
- 纯商业分析文档
